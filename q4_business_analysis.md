# ข้อ 4: ManufacturePlus Scenario

## 1. Functional Requirements (3 ข้อเพื่อลดการเดินมาถาม HR เรื่องวันลา)

เพื่อแก้ปัญหาพนักงาน 800 คนเดินมาถาม HR ตลอดเวลา ระบบควรมีฟังก์ชันดังต่อไปนี้:
1. **Employee Self-Service Dashboard (ระบบเช็คสิทธิ์ด้วยตัวเอง):** หน้าจอ Web/Mobile App ที่พนักงานสามารถล็อคอินเข้าไปดูโควตาวันลาทั้งหมด (ลาป่วย, ลากิจ, ลาพักร้อน) ทั้งจำนวนที่ใช้ไปแล้วและจำนวนที่เหลืออยู่ได้แบบ Real-time
2. **Automated Leave Workflow (ระบบลางานและอนุมัติออนไลน์):** ฟังก์ชันให้พนักงานกดส่งคำขอลาผ่านระบบ โดยส่งแจ้งเตือน (เช่น อีเมล หรือ LINE) ไปยังหัวหน้างานเพื่อกดอนุมัติ เมื่ออนุมัติแล้วระบบจะตัดยอดวันลาให้อัตโนมัติ โดยไม่ต้องผ่าน HR
3. **Monthly Leave Summary Notification (ระบบแจ้งเตือนสรุปวันลา):** บอทหรือระบบส่งข้อความแจ้งเตือนสรุปโควตาวันลาคงเหลือให้พนักงานทุกคนอัตโนมัติ (เช่น ผ่าน LINE OA ของบริษัท) ทุกๆ สิ้นเดือน หรือแจ้งเตือนเมื่อโควตาใกล้หมด

## 2. เทคนิค Data Cleansing (กรณีชื่อผิด / Employee ID ซ้ำจากการใช้ Excel)
ในการย้ายข้อมูล (Data Migration) จาก Excel 10 ปีเข้าระบบใหม่ จะใช้เทคนิคดังนี้:
1. **การจัดการ Employee ID ซ้ำ (Duplicate Resolution):** * **เทคนิค:** ใช้ `National ID` (เลขบัตรประชาชน 13 หลัก) หรืออีเมล เป็น Primary Truth (แกนหลักในการอ้างอิง) หากพบ Employee ID ซ้ำกัน ให้ตรวจสอบจากเลขบัตรประชาชน หากเลขบัตรต่างกันแปลว่าเป็นคนละคน ให้ระบบ Generate Employee ID ใหม่ให้คนที่ซ้ำ (เช่น เติม Suffix -A, -B) แล้วอัปเดตกลับไป 
2. **การจัดการชื่อพิมพ์ผิด (Data Standardization & Fuzzy Matching):**
   * **เทคนิค Standardization:** เขียน Script (เช่น Node.js หรือ Python) เพื่อ Trim ช่องว่างหัวท้าย และลบอักขระพิเศษออกก่อน
   * **เทคนิค Fuzzy Matching:** หากมีชื่อที่คล้ายกันมาก (เช่น "สมชาย" กับ "สมช่าย") ใช้ Algorithm วัดความคล้ายของตัวอักษร (เช่น Levenshtein Distance) จัดกลุ่มชื่อที่น่าจะผิดมาให้ HR ตรวจสอบเทียบกับสำเนาบัตรประชาชนตัวจริง ก่อน Import ลง Database
